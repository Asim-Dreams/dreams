Advanced Functions

 

Item Window Synchronization

When you click on an event (to highlight it) the Item window tries to find the correlated item in the "Item Window" and then select it (automatically ensures it is visible).
The reverse synchronization can be also performed: select an item in the Item Window and them hit Return or Space key (or double click). 2Dreams will look for the first appearance of the selected item and will shade the cycle and highlight the item. The events window is automatically scrolled to ensure the item is visible.

Note: If you want to avoid automatic synchronization when highlighting events, hold Shift key while clicking.

 

Highlighting

2Dreams has the ability of emphasize (highlight) events by some criterion. The criterion must be expressed in terms of <DRAL-Tag, value> pairs.
Visually 2Dreams selects some light colour (typically white) to draw the border of highlighted events.  The  <DRAL-Tag, value> pairs are given clicking events on screen.

Basic Highlighting

To highlight events by a given criterion you must first choose a DRAL-Tag. Look for "tools" bar and select a Tag from the list (Figure 1). If you need a Tag that is not in the list just type its name in the edit space. To select a value just click (left button) in the target event, the DRAL-Tag value for the selected event is checked and all events with the same pair <DRAL-Tag, value> will be highlighted (Figure 2). To disable the highlight click in any empty area of the events window or click on "Disable Last" entry of Tools/Highlight menu.

Additive Highlighting

You can keep highlighted events by more than one criterion at the same time (Figure 3). In fact, 2Dreams allows you only one DRAL-Tag at once but multiple values for this Tag. To achieve this functionality you need to hold CTRL while clicking the events and the new tag value will be added to a stack of criteria. Different colours will be used to each criteria. If you click in an empty area of the events window or click on "Disable Last" entry of Tools/Highlight menu, the last pushed criterion will be popped. To remove all the criteria at once you can use the menu option Tools/Highlight/Disable All.

When several criteria are active, you can check them out using the menu option Tools/Highlight/Show Criteria (Figure 4).

Relationship Scanning

 2Dreams can scan for relationships between different items. This can be useful to find instructions parents, relations between micro and macro instructions, merged memory transactions, etc. The scanning algorithm works with any set of values tag. The default used tag is ANCESTOR. To start up a scanning over a given item right-click on the event and select “Relationship Scan…” (Figure 5).

Figure 5

 After selecting this option if the clicked event has the tracking tag (ANCESTOR by default), the items referenced by such tag will be scanned. Both the clicked item and the referenced ones will be automatically highlighted. If you activate the "Relationship Window" (right bottom corner in Figure 6), you will be able to manage the correlated items. The Relationship Window is split into two vertical sections. In the left frame, you have a tree view of the item respect to the scanned tag. The right frame shows the most common tags (check preferences) for the currently selected item in the left frame.

Figure 6

Once the initial scan was performed, you can manage the correlated items right-clicking in the left frame of the Relationship Window. As shown in Figure 7 you can insight in the relationship scanning the same tag for any correlated item (Expand Node), the new scanned items will be automatically highlighted and added to the tree view (Figure 8).  You can also switch on and off the highlighting and the arrows on the items or clean up the whole scan (Reset Scan).
