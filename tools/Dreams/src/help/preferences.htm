2Dreams Preferences


2Dreams is customizable in several ways. To access the preferences dialogs click on Tools/Preferences. You will find four tabs (TAGS, DBase, File and General) in the dialog. In the following sections, we describe the functionality of each of those tabs. Some preferences are saved and updated automatically. For instance, the position and size of the different windows and toolbars are tracked by default. If for some reason you need to change by hand some of those properties edit the file ~/.qt/2dreamsrc in Unix systems or look for BSSAD/2Dreams prefix in the register in Windows systems.

TAGS Section

In the TAGS Section, you can manage the Favourite Tag List and the Relationship tracking Tag. The Favourite Tags are used in the Tag Window (right bottom corner of Figure 1) if "Show Only Favorite Tags" is enabled (see Behaviour Section) and in the Highlight criterion selector (top toolbar). The Relationship tracking tag is used to track for relations between items (see Advanced Features). The zoom icon is active only when an event file is loaded and allows you to select one of the used tags in the event file. Bear in mind that the selected tag must be of the type set of values and each value must be a valid item identifier.

DBase Section

In the DBase section, you will find four sliders to change the amount of space (memory) used for the internal database. The maximum number of cycles is self-explanatory. The Tag Heap and Mutable Tag Heap sizes can be increased if the number of items or the amount of related information (tags) is higher than the expected (you will get an error message in this case when loading the event file). The symbol table size limits the number of string tags (like disassembly).

File Section

In the File Section, you can change the location of the log file used by 2Dreams and specify a default ADF (see What's an ADF). You can also enable a DTD checking. Note that 2Dreams is not able to perform such a checking so you will need some external tool (like xmllint). Web connectivity to intranet server aforge.shr.intel.com is also required to access the DTD.

Appearence Section

In this section, you will find some common appearance settings (background color, font size, etc) and grid lines configuration. Grid lines are mainly implemented for some planified features extensions.

Behaviour Section

In this section you have two check boxes. The Automatic purge option, if enabled, forces the "delete item" command of all the live items when the end of the events
file is reached (or loading stoped by user). If the Tag Window option is cheked only the Favorite Tags (see TAGS Section)will be shown in the Tag Window, otherwise all the
item tags are shown.
